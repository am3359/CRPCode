所有任务一次创建完成，每个任务是一个状态机，有定时复位机制
运动控制命令是一个字符串列表，分为阻塞式和非阻塞式。单独的命令字符串用“(”开头，“)”结束为阻塞执行，需要等到该命令完成再返回。用“[”开头，“]”结束，中间多个命令字符串用“;”分隔，为非阻塞执行，所有命令同时发出，不等待立即返回。
命令字符串：阀（4字节）：Vnnb；步进电机（4字节）：Snnc；蠕动泵（8字节）：Pnnktttt；旋转泵（8字节）：Rnnctttt；温度（4字节）：Tnnb。编号nn=[01~99]；开关b=[0/1]；速度c=[0/1/2]；速度和方向k=[0/10]+[0/1/2]；tttt=[0001~9999] x 0.1秒
任务：任务之间要传递信息 
Main：硬件初始化，创建主任务start_task，启动调度器
主任务：(start_task) 创建软件定时器，消息队列，信号量，事件标志组，任务通知等，创建完成删除自己。
HMI界面UART通讯：(hmi_task)负责界面显示。发送命令给HMI界面，接收HMI界面消息，接收其它任务的消息

串口通讯：(com_task)接收控制命令，返回信息。接收串口消息，发送消息给串口

阀控制、步进电机控制、泵控制：(valve_task，stepmoto_task，pump_task)接收阀控制信息（开关），接收步进电机控制信息（速度，位置），接收泵控制信息（速度，方向，时间）；返回各硬件状态
